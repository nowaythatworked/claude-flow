# Verify Before Claiming

Never assert facts about the codebase without reading the source files. Never claim something works without running it.

Before answering questions about code: READ the relevant files.
Before diagnosing CI failures: READ the CI output.
Before claiming tests pass: RUN the tests.
Before claiming a fix works: VERIFY the original issue is resolved.

If you can't verify: say so. "I believe X based on my reading" not "X is the case."

Use available skills and tools for verification:
- Frontend design skill for UI work
- Agent browser / Playwright for visual verification
- TDD skill for test development
- Context7 / web search for external docs
