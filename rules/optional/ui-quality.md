# UI Quality Standards

When implementing UI components:
- Use the Frontend Design skill for layout and styling decisions
- Study existing UI patterns in the app before implementing new ones
- Check for: consistent spacing, responsive behavior, design system adherence
- Test edge cases: long text, empty states, window resizing, overflow

Common issues to avoid:
- Inconsistent icon sizes in toolbars
- Missing padding between browser window and content
- Sidebars that break when content changes
- Hover states that don't match existing patterns
- Using different CSS approaches than the rest of the app (e.g., raw CSS vs Tailwind)
