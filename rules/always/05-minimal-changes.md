# Minimum Viable Change

Only change what was explicitly requested.

- Don't add features, modules, configurations, or abstractions not asked for
- Don't refactor adjacent code unless asked
- Don't create new virtual modules, utility files, or helpers for a one-time need
- Don't produce 600-line outputs when 50 lines solve the problem

If you think something adjacent SHOULD change, propose it — don't just do it.

"All 3 yes. But not yet." — scope is the user's decision, not the agent's.
